# mono-ledger-sync v1.0.0

The `1.0.0` release completes the public-ready local Monobank personal finance
workspace. It keeps the established product boundary: one user's personal token,
a loopback-only Fastify server, profile-scoped SQLite data, deterministic local
workflows, and no hosted relay.

## Added

- A clearly labeled first-run demo backed by synthetic fixtures. Activating a
  valid Monobank token clears demo ledger rows before switching to live sync.
- Native secure token persistence through macOS Keychain Services, Windows
  Credential Manager/Credential Locker, and Linux Secret Service, with explicit
  session-only fallback.
- Per-account sync-run details for completed accounts, failed statement windows,
  retry timing, and user-facing failure reasons through migration
  `0027_sync_run_details`.
- Complete budget setup with month, currency, category, historical suggestions,
  target limit, rollover, copy-forward, and close/reopen actions.
- Recurring dashboard summaries, suggestion decisions, manual streams, detail
  history, missed dates, and linked transaction filters.
- README quick start, fixture-safe screenshots, migration/recovery guidance,
  accessibility baseline, public release checklist, localization decision, and
  v1 support boundary.

## Changed

- Transactions keep memoized row rendering and now have a measured 10,000-row
  pagination guardrail.
- Browser route smoke covers all 13 routes and checks accessible control names,
  form labels, duplicate IDs, image alternatives, route announcements, console
  errors, overlays, and favicon availability.
- Monobank-specific account types use bilingual English/Ukrainian labels while
  the v1 interface remains English.
- Public documentation and screenshots are included in the npm package.

## Security and privacy

- Token values are sent to platform credential adapters through stdin, never
  command arguments, SQLite, exports, diagnostics, or logs.
- Demo and live transaction rows cannot be mixed during normal first-run
  promotion.
- Partial-sync details contain local account/window metadata and redacted error
  messages, not raw payloads or token material.
- The logging and repository secret scans remain release gates.

## Upgrade notes

This is a major version bump from `0.4.0` to `1.0.0`, but it does not remove a
package export or require a new data directory. Existing databases migrate
forward automatically by adding the optional `sync_runs.details_json` column.
Create a local backup before upgrading, then verify database integrity and run a
normal sync.
