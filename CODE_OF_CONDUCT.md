# Code of Conduct

## Scope

This code of conduct applies to project spaces for `mono-ledger-sync`, including issues, pull requests, discussions, documentation, and any maintainer-led community channels.

## Standards

Contributors are expected to:

- Be respectful and direct when discussing technical work.
- Keep feedback focused on code, documentation, product behavior, and security impact.
- Assume good intent while still being clear about risks, regressions, and tradeoffs.
- Respect user privacy and avoid sharing real banking tokens, account identifiers, statements, or sensitive financial data.
- Help keep the project useful for people who want local ownership of their financial data.

## Unacceptable behavior

Unacceptable behavior includes:

- Harassment, intimidation, threats, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without explicit permission.
- Sharing credentials, real financial records, or other sensitive data in public project spaces.
- Repeatedly disrupting technical discussion after maintainers have asked for a change in behavior.

## Enforcement

Maintainers may remove comments, close issues, block users, or take other reasonable action when behavior violates this code of conduct.

Report conduct concerns privately through the maintainer contact listed in [SECURITY.md](SECURITY.md). If a report includes security-sensitive details or personal financial data, do not open a public issue.
